Microsoft Intune Autopilot Deployment Guide

This guide walks through deploying Windows 11 devices using Microsoft Intune and Windows Autopilot, from hardware hash collection through first user sign-in.

Prerequisites

  • Microsoft Intune subscription (standalone or M365 E3/E5)
  • Azure AD Premium P1 or P2
  • Windows 11 Pro, Enterprise, or Education
  • Internet connectivity during OOBE
  • DNS resolution for login.microsoftonline.com and enterpriseregistration.windows.net

Step 1: Collect Hardware Hashes

Each device needs its hardware hash registered with Autopilot before deployment.

Option A: PowerShell (existing devices)

Install-Script -Name Get-WindowsAutopilotInfo
Get-WindowsAutopilotInfo -OutputFile C:\AutopilotHashes.csv

Option B: OEM Registration

Dell, HP, and Lenovo can register hardware hashes at time of purchase. Contact your OEM account representative to enable this.

Step 2: Create Deployment Profile

  1. Navigate to Intune > Devices > Windows > Windows enrollment > Deployment Profiles
  2. Click Create profile > Windows PC
  3. Configure OOBE settings:
    • Deployment mode: User-Driven or Self-Deploying
    • Join type: Azure AD joined (recommended) or Hybrid Azure AD
    • Skip privacy settings, EULA, and account change
  4. Assign to a device group containing your Autopilot-registered devices

Step 3: Configure Enrollment Status Page

The ESP ensures all policies and apps are installed before the user reaches the desktop.

  • Navigate to Intune > Devices > Windows > Windows enrollment > Enrollment Status Page
  • Set timeout to 60 minutes minimum
  • Block device use until all required apps are installed
  • Only track installation for required apps (not available apps)
Note: If ESP takes longer than 60 minutes, check your network bandwidth and app package sizes. Win32 apps typically install faster than MSI packages during ESP.

Step 4: Assign Apps and Policies

Before deployment, ensure these are assigned to the device group:

  • Compliance policy (require BitLocker, Windows Defender, OS version)
  • Configuration profiles (Wi-Fi, VPN, certificates)
  • Required apps (Microsoft 365 Apps, LOB apps, Defender for Endpoint)
  • Windows Update rings

Step 5: Deploy and Validate

  1. Power on the device and connect to the internet
  2. The Autopilot profile should apply automatically at the OOBE screen
  3. User signs in with their Azure AD credentials
  4. ESP runs, installing all assigned apps and policies
  5. Validate in Intune: Devices > Windows > [device] > Overview

Troubleshooting

  • Profile not applying: Check that the hardware hash was imported and the device is in the correct group
  • ESP timeout: Review app installation logs at C:\ProgramData\Microsoft\IntuneManagementExtension\Logs
  • Hybrid join failure: Verify the Intune Connector for Active Directory is running and has connectivity
