Zero Trust Identity Assessment Checklist

Comprehensive identity verification and access control assessment framework for enterprise security teams. 2026 Edition — aligned with NIST SP 800-207, Microsoft Zero Trust, and CISA Zero Trust Maturity Model v2.0.

This assessment checklist helps security teams evaluate Zero Trust maturity across six pillars: Identity, Devices, Networks, Applications, Data, and Visibility. Use this checklist to identify gaps and prioritize remediation efforts.

1. Identity Pillar Assessment

Identity Verification Controls

Authentication Strength

All users required to use MFA — no exceptions for legacy apps or service accounts Optimal
Phishing-resistant MFA (FIDO2/Windows Hello) deployed for privileged accounts Optimal
SMS and voice OTP disabled as MFA methods Advanced
Legacy authentication protocols blocked via Conditional Access Advanced
Passwordless sign-in enabled where supported Optimal

Identity Risk Management

Microsoft Entra ID Protection enabled with sign-in risk policy Advanced
User risk policy configured to require password change on high risk Advanced
Continuous Access Evaluation (CAE) enabled for supported apps Optimal

Privileged Identity

PIM activated for all privileged roles with just-in-time access Advanced
Standing Global Administrator assignments eliminated (0 persistent admins) Optimal
Emergency access accounts in place, tested quarterly Advanced

2. Device Pillar Assessment

Device Trust Controls

Device Registration and Compliance

All corporate devices enrolled in Microsoft Intune Advanced
Device compliance policies enforced as Conditional Access grant control Advanced
Compliant device status continuously verified (not just at sign-in) Optimal
BitLocker enabled on all Windows endpoints with key escrow in Entra ID Advanced

Endpoint Security

Microsoft Defender for Endpoint deployed and reporting to Intune Advanced
Threat and vulnerability management configured with remediation SLAs Optimal
Attack surface reduction rules enabled in block mode Advanced

3. Network Pillar Assessment

Network Segmentation Controls

Perimeter and Segmentation

VPN replaced or supplemented with Microsoft Entra Private Access (ZTNA) Optimal
Network micro-segmentation implemented for sensitive workloads Optimal
Named locations defined in Conditional Access for known-good networks Advanced
DNS filtering deployed (Microsoft Defender for DNS or equivalent) Advanced

4. Applications Pillar Assessment

Application Access Controls

App Registration Hygiene

All app registrations reviewed — stale apps removed Advanced
Admin consent required for all new OAuth app permissions Advanced
Application access policies restrict which users can consent to apps Advanced

Access Control

Conditional Access policies scoped per application (not global only) Optimal
Microsoft Defender for Cloud Apps deployed for SaaS visibility Advanced
Session controls enabled for sensitive applications Optimal

5. Data Pillar Assessment

Data Protection Controls
Sensitivity labels deployed and enforced across M365 Advanced
DLP policies active for PII, financial, and health data types Advanced
External sharing disabled by default in SharePoint/OneDrive Advanced
Customer Lockbox enabled for Microsoft support access Advanced
Data lifecycle management policies configured for retention and deletion Optimal

6. Visibility and Analytics Pillar Assessment

Security Operations Controls
Entra ID sign-in and audit logs streamed to Microsoft Sentinel or equivalent SIEM Advanced
Audit log retention minimum 90 days (recommended: 365 days) Advanced
XDR solution deployed correlating endpoint, identity, and email signals Optimal
UEBA (User and Entity Behavior Analytics) enabled for insider threat detection Optimal
SOC playbooks defined for common Entra ID incident types Advanced
Assessment Scoring: Count the number of checked items across all pillars. 0-30%: Traditional tier — significant Zero Trust gaps. 31-70%: Advanced tier — good foundation, key gaps remain. 71-100%: Optimal tier — strong Zero Trust posture.

Last updated: March 2026. Maintained by Grome Security Research.

This assessment checklist aligns with NIST SP 800-207 (Zero Trust Architecture), CISA Zero Trust Maturity Model v2.0, Microsoft Zero Trust deployment guidance, and CIS Controls v8. The maturity model tiers reflect Microsoft's Zero Trust deployment guidance levels: Traditional, Advanced, and Optimal.
